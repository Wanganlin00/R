3.1 函数组件

  1. formals() 控制如何调用函数的参数列表
  2. body() 函数内部的代码
  3. environment() 确定函数如何查找与名称相关联的值的数据结构

source reference 即 srcref属性，指向用于创建函数的源代码

Code
f02 <- function(x, y) {
  # A comment
  x + y
}

formals(f02)
#> $x
#> 
#> 
#> $y
body(f02)
#> {
#>     x + y
#> }
environment(f02)
#> <environment: R_GlobalEnv>

attributes(f02)
#> $srcref
#> function(x, y) {
#>   # A comment
#>   x + y
#> }

3.2 Primitive functions

函数有三个组件的规则有一个例外，Primitive functions ( 如sum()) 直接调用C code。

Primitive functions有内置builtin 和 特殊special 类型。

Code
sum
#> function (..., na.rm = FALSE)  .Primitive("sum")
typeof(sum)
#> [1] "builtin"

`[`
#> .Primitive("[")
typeof(`[`)
#> [1] "special"

这些函数主要存在于 C 中，而不是 R 中。Primitive functions仅在 base package 找到。

Code
formals(sum)
#> NULL
body(sum)
#> NULL
environment(sum)
#> NULL

3.5 词法作用域

函数的作用域（scope）在R中称为环境。

对象可以通过参数传递到函数中，但不会被函数改变。传递的是对象的副本而不是对象本身。除非使用 <<- 特殊赋值符。

Code
x <- 2
y <- 3
z <- 4
f <- function(w){
    z <- 2
    x <- w*y*z
    return(x)
}
f(x)
#> [1] 12
x
#> [1] 2
y
#> [1] 3
z
#> [1] 4

3.5.1 Name masking

在函数内部定义的名称会掩盖在函数外部定义的名称。

Code
x <- 10
y <- 20
g02 <- function() {
  x <- 1
  y <- 2
  c(x, y)
}
g02()
#> [1] 1 2

如果未在函数中定义名称，则 R 将向上查找。

Code
x <- 2
g03 <- function() {
  y <- 1
  c(x, y)
}
g03()
#> [1] 2 1

y
#> [1] 20

3.6 惰性求值

Lazy evaluation 是一种将操作延迟到真正需要它的结果时才执行的优化技术。

惰性求值由称为 promise 或thunk 的数据结构提供支持。

3.6.1 promise

一个表达式，调用函数的环境，一个值

Code
double <- function(x) { 
  message("Calculating...")
  x * 2
}

h03 <- function(x) {
  c(x, x)
}

h03(double(20))
#> Calculating...
#> [1] 40 40

3.7 传递参数

关键字传参，位置传参

3.7.1 默认参数

Code
h04 <- function(x = 1, y = x * 2, z = a + b) {
  a <- 10
  b <- 100
  
  c(x, y, z)
}
h04()
#> [1]   1   2 110

# 查看参数的名称和默认值
args(h04)
#> function (x = 1, y = x * 2, z = a + b) 
#> NULL

3.7.2 缺失参数

Code
h06 <- function(x = 10) {
  list(missing(x), x)
}
str(h06())
#> List of 2
#>  $ : logi TRUE
#>  $ : num 10

str(h06(10))
#> List of 2
#>  $ : logi FALSE
#>  $ : num 10

3.8 特殊参数...

variable arguments，有了它，函数可以接受任意数量的附加参数。还可以将附加参数传递给另一个函数

Code
i01 <- function(y, z) {
  list(y = y, z = z)
}

i02 <- function(x, ...) {
  i01(...)
}
str(i02(x = 1, y = 2, z = 3))
#> List of 2
#>  $ y: num 2
#>  $ z: num 3

特殊形式 ..N，按位置引用参数

Code
i03 <- function(...) {
  list(first = ..1, third = ..3)
}
str(i03(1, 2, 3))
#> List of 2
#>  $ first: num 1
#>  $ third: num 3

list(...)计算参数并将它们存储在列表中

Code
i04 <- function(...) {
  list(...)
}
str(i04(a = 1, b = 2))
#> List of 2
#>  $ a: num 1
#>  $ b: num 2

3.9 退出函数

返回一个对象，或者，错误

3.9.1 隐式返回值

最后计算的表达式是返回值

Code
j01 <- function(x) {
  if (x < 10) {
    0
  } else {
    10
  }
}
j01(5)
#> [1] 0

j01(15)
#> [1] 10

3.9.2 显示返回值

调用return()

Code
j02 <- function(x) {
  if (x < 10) {
    return(0)
  } else {
    return(10)
  }
}

3.9.3 不可见值

大多数函数都以可见方式返回：在交互式上下文中调用函数会自动打印结果

Code
j03 <- function() 1
j03()
#> [1] 1

通过应用invisible()于最后一个值来阻止自动打印：

Code
j04 <- function() invisible(1)
j04()

若要验证此值是否确实存在，可以显式打印它或将其括在括号中,或者使用返回值和可见性标志：withVisible()

Code
print(j04())
#> [1] 1


(j04())
#> [1] 1

withVisible(j04())
#> $value
#> [1] 1
#> 
#> $visible
#> [1] FALSE

不可见返回的最常见函数是：<-

Code
a <- 2
(a <- 2)
#> [1] 2

a <- b <- c <- d <- 2

3.9.4 Errors

如果函数无法完成其分配的任务，则应抛出错误，并立即终止函数的执行。

Code
j05 <- function() {
  stop("I'm an error")
  return(10)
}
j05()
#> Error in j05(): I'm an error

3.9.5 退出处理程序

Code
j06 <- function(x) {
  cat("Hello\n")
  on.exit(cat("Goodbye!\n"), add = TRUE) # 使用  add = TRUE 时始终设置
  
  if (x) {
    return(10)
  } else {
    stop("Error")
  }
}

j06(TRUE)
#> Hello
#> Goodbye!
#> [1] 10


j06(FALSE)
#> Hello
#> Error in j06(FALSE): Error
#> Goodbye!

3.10.1 前缀形式

是 R 代码中最常见的形式，参数按确切名称匹配，然后与唯一前缀匹配，最后按位置匹配。

Code
k01 <- function(abcdef, bcde1, bcde2) {
  list(a = abcdef, b1 = bcde1, b2 = bcde2)
}
str(k01(2, 3, abcdef = 1))
#> List of 3
#>  $ a : num 1
#>  $ b1: num 2
#>  $ b2: num 3
str(k01(2, 3, a = 1))  # 部分匹配
#> List of 3
#>  $ a : num 1
#>  $ b1: num 2
#>  $ b2: num 3
options(warnPartialMatchArgs = TRUE)
x <- k01(a = 1, 2, 3)
#> Warning in k01(a = 1, 2, 3): partial argument match of 'a' to 'abcdef'

3.10.2 中缀形式

Code
`%+%` <- function(a, b) paste0(a, b)
"new " %+% "string"
#> [1] "new string"

3.10.3 替换形式

Code
`second<-` <- function(x, value) {
  x[2] <- value
  x
}
x <- 1:10
second(x) <- 5L
x
#>  [1]  1  5  3  4  5  6  7  8  9 10
